# Workspace

## Overview

pnpm workspace monorepo using TypeScript. Each package manages its own dependencies.

## Stack

- **Monorepo tool**: pnpm workspaces
- **Node.js version**: 24
- **Package manager**: pnpm
- **TypeScript version**: 5.9
- **API framework**: Express 5
- **Database**: PostgreSQL + Drizzle ORM
- **Validation**: Zod (`zod/v4`), `drizzle-zod`
- **API codegen**: Orval (from OpenAPI spec)
- **Build**: esbuild (CJS bundle)

## Key Commands

- `pnpm run typecheck` — full typecheck across all packages
- `pnpm run build` — typecheck + build all packages
- `pnpm --filter @workspace/api-spec run codegen` — regenerate API hooks and Zod schemas from OpenAPI spec
- `pnpm --filter @workspace/db run push` — push DB schema changes (dev only)
- `pnpm --filter @workspace/api-server run dev` — run API server locally

See the `pnpm-workspace` skill for workspace structure, TypeScript setup, and package details.

## AFMOD Artifact (`artifacts/afmod`)

Arabic RTL ministry portal "وزارة اللوجستيك — Arab First City" with Discord auth. **Pure vanilla HTML / CSS / JavaScript** — no React, no TypeScript, no Tailwind, no build framework. Vite is used solely as a dev/preview static file server (no transpilation since all source is plain JS).

- **File structure** (only HTML/CSS/JS — no other languages):
  - `index.html` — single-page shell with login view + home view (3 tab panels) + gallery dialog + car dialog.
  - `styles.css` — full design system: navy/gold tokens, tabs bar, filter pills, house cards, car cards, gas station cards, dialogs.
  - `houses.js` — `window.HOUSES` dataset (6 houses) and `window.formatPrice` helper.
  - `cars.js` — `window.CARS` dataset (93 ERLC vehicles) and `window.CAR_BODIES` filter list.
  - `gas-stations.js` — `window.GAS_STATIONS` dataset (3 stations: City, County, Springfield).
  - `app.js` — IIFE handling auth, tabs, dashboard rendering, dialogs, image fallback, purchase events. No module imports.
  - `public/` — static assets.
  - `vite.config.js` — minimal: just sets PORT/BASE_PATH/outDir for static serving.
  - `vercel.json` — Vercel hosting config (static serve + cache headers + `/api/*` rewrite to api-server).
- **Tabs**: home view has three sticky tabs — السكن (houses), المعارض و السيارات (cars, with body-type filter pills), محطات الوقود (gas stations).
- **Auth flow**: user enters Discord username → `POST /api/auth/request-code` → bot DMs 6-digit code (10 min TTL, 30s cooldown) → `POST /api/auth/verify-code` → signed-cookie session (7d). On load, `app.js` calls `GET /api/auth/me` and shows login view on 401, home view on success.
- **Required env**: `DISCORD_BOT_TOKEN`, `DISCORD_GUILD_ID`, `SESSION_SECRET`, `AFMOD_BOT_URL`, `AFMOD_BOT_API_KEY` (api-server). Discord bot must have **SERVER MEMBERS INTENT** enabled.
- **Anti-leak watermark**: 30-tile grid of user avatar + display name + @username, diagonal, low opacity.
- **Theme**: navy `#0a1f4d` / royal gold `#d4af37` / ivory `#fcfcf7`. Display: Reem Kufi, body: Noto Kufi Arabic, serif: Playfair Display.
- **Image fallback**: every wiki image uses `referrerpolicy="no-referrer"` + `onerror="window.__afmodImageFallback(this)"` which retries once with cache-bust then swaps to inline SVG placeholder — fixes flaky CDN loads on weak connections.

## AFMOD Showroom Bot (`bot/`)

Standalone Python bot (discord.py + Flask) designed for **bot-hosting** (always-on Python). Receives purchase events from the website and integrates with the existing AFMOD bot suite (bank bot, phone bot, car bot).

- **Files**: `bot.py`, `requirements.txt`, `catalog.json` (auto-generated), `README.md`.
- **Catalog**: regenerated by `node scripts/build-catalog.mjs` from `cars.js` + `houses.js` so the bot always re-checks prices server-side (website is never trusted on price).
- **Flow**: website → `POST /api/showroom/buy-car` (api-server, requires session cookie) → forwards with `X-AFMOD-API-Key` to bot's `POST /events/buy-car` → bot checks bank balance → deducts → asks phone bot to spawn the car → DMs the citizen a receipt. Auto-refunds if spawn fails.
- **Required env**: `AFMOD_BOT_API_KEY`, `DISCORD_BOT_TOKEN`, `BANK_BOT_BASE_URL`, `BANK_BOT_API_KEY`, `PHONE_BOT_BASE_URL`, `PHONE_BOT_API_KEY`, `AFMOD_BOT_PORT` (default 8787).

## Hosting topology (Vercel full-stack)

The entire project (frontend + backend) deploys to **Vercel** as a single project from the repo root. `bot.py` is separate and must be hosted on a Python bot-hosting service.

- **Frontend** → Vercel static (built from `artifacts/afmod`, output to `artifacts/afmod/dist/public`).
- **API Server** → Vercel Serverless Function (`api/index.js` → `artifacts/api-server/dist/handler.mjs`). All `/api/*` routes are handled here.
- **Bot** → `bot.py` hosted separately on a Python bot-hosting service (e.g. bothosting.net). Must be reachable at a public URL set as `AFMOD_BOT_URL` env var.

## Vercel Environment Variables Required

Set these in Vercel Dashboard → Project → Settings → Environment Variables:

| Variable | Description |
|---|---|
| `DATABASE_URL` | Neon PostgreSQL connection string |
| `DISCORD_BOT_TOKEN` | Discord bot token (must have SERVER MEMBERS INTENT) |
| `DISCORD_GUILD_ID` | `1183042468854382612` |
| `SESSION_SECRET` | Random string ≥32 chars (e.g. openssl rand -hex 32) |
| `AFMOD_BOT_URL` | Public URL of bot.py host (e.g. `https://yourdomain.bothosting.net`) |
| `AFMOD_BOT_API_KEY` | Shared secret key between api-server and bot.py |
